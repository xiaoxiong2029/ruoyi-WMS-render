# Render deployment

This project runs on Render as:

- `ruoyi-redis`: Render Key Value service.
- `ruoyi`: Docker web service for the Spring Boot backend. For no-card demos,
  this container also starts a local MariaDB instance and imports `wms.sql`.
- `ruoyi-vue`: Static Site for the Vue frontend.

The root `render.yaml` Blueprint creates `ruoyi-redis`, `ruoyi`, and
`ruoyi-vue` together. Redis is injected automatically into the backend.

## Backend service

Use these settings if creating the service manually:

- Runtime: Docker
- Root Directory: `wms-ruoyi`
- Dockerfile Path: `./Dockerfile`

Environment variables:

```text
SPRING_PROFILES_ACTIVE=render
TZ=Asia/Shanghai
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_DATABASE=ry_wms
MYSQL_USER=ruoyi
MYSQL_PASSWORD=<generated by Blueprint>
REDIS_URL=<injected by Blueprint>
```

Render injects `PORT` automatically. If you deploy from the root `render.yaml`
Blueprint, Render creates the Key Value service and fills the backend cache
variable automatically.

## Frontend static site

- Root Directory: `ruo-yi-wms-vue`
- Build Command: `npm ci && npm run build:prod`
- Publish Directory: `dist`

Environment variables:

```text
VITE_APP_BASE_API=https://<your-backend-service>.onrender.com
VITE_APP_CONTEXT_PATH=/
```

Add this rewrite rule for Vue Router:

```text
Source: /*
Destination: /index.html
Action: Rewrite
```

## Database

The demo Docker image imports `wms-ruoyi/script/sql/wms.sql` automatically on
first startup. On Render free services this database is ephemeral and can reset
when the container is recreated. Use a paid persistent MySQL service before
storing real warehouse data.
